Terceira Etapa

  1. Criando as rotas para as views

    1. Criando arquivos urls.py dentro de cada app
      • Dentro do arquivo pagina/urls.py importamos o módulo django.urls e chamamos a função path.
      • Importamos as views que queremos passar às rotas.
      • Usamos uma variável padrão do Django chamada urlpatterns, que recebe uma lista. Dentro dessa lista, chamamos a função path, que deve receber uma string com a rota desejada, conforme a documentação oficial, e a função da view.
    2. Definindo a URLconf.
      • Na pasta do projeto principal praticando, no arquivo urls.py, devem ser incluídas as URLs das views.
      • Primeiro, no mesmo módulo padrão para path, chamamos a função include.
      • Observe que a lista urlpatterns está presente nesse arquivo e que já temos definida a rota admin.
      • Criamos a rota para a página inicial do site, usando uma string vazia para o primeiro parâmetro e logo em seguida utilizamos a função include, que recebe, em formato de string, o nome do app e o nome do arquivo urls → "pagina.urls".
      • Criamos também a rota para o app "etapas/", e passamos include como segundo parâmetro de path com "etapas.urls", garantindo o mapeamento correto das rotas.

  2. Ativo os apps e confirmo o funcionamento

    1. Na pasta do projeto, buscamos o arquivo settings.py e, dentro da lista armazenada na constante INSTALLED_APPS, adicionamos os nomes dos apps criados dentro de uma string, no caso pagina e etapas.
    2. Usamos o terminal shell para iniciar o servidor de teste.

      3. Usando os seguintes comandos:

      • CTRL + P
      • venv/Scripts/activate
      • python manage.py runserver
    3. Acesso o navegador e, na barra de URL, escrevo: http://127.0.0.1:8000/
      • Testo a rota inicial
      • E as rotas de etapas
      • http://127.0.0.1:8000/etapas/1
      • http://127.0.0.1:8000/etapas/2
      • http://127.0.0.1:8000/etapas/3
