  1. Criei uma pasta global para todos os arquivos HTML

    1. Criei uma pasta chamada templates
      • Dentro de templates, criei uma pasta global.
      • Dentro de global, criei duas pastas, uma para cada app.
    2. Criei um arquivo HTML na pasta global/pagina
      • Esse arquivo, inicialmente, vai receber uma lista de tarefas/condições para a execução do projeto.
      • O arquivo se chama inicio.html.
    3. Criei um arquivo HTML na pasta global/etapas
      • Ele vai receber os arquivos HTML (texto informativo) do passo a passo de como cada parte está sendo executada.
      • Futuramente, haverá vários arquivos nomeados como etapa(n), que representarão o total de etapas e conhecimentos aplicados no projeto.
    4. Definindo a pasta templates como global no Django
      • No arquivo settings.py do projeto "praticando", dentro da constante TEMPLATES, que é uma lista de dicionários, devemos configurar a chave 'DIRS', que recebe uma lista contendo BASE_DIR / 'templates', indicando o diretório onde os templates estão armazenados.

  2. Criei as Views de ambos os apps

    1. Criei a view do app página
      • Dentro do arquivo views.py, iniciei a função paginaInicial.

        • Ela recebe uma requisição (request) como parâmetro e retorna a função render, que precisa receber o que está sendo requisitado como primeiro parâmetro e o caminho do HTML que será renderizado, nesse caso -> global/pagina/inicio.html.

    2. Criando a View do app etapas
      • O processo é o mesmo da parte 2.a, porém com duas funções: etapa1 e etapa2, além de quantas forem necessárias. O total de arquivos aumentará conforme a quantidade de passos. O caminho passado segue o seguinte padrão -> global/etapas/etapaX.html (onde X representa a etapa do processo com números crescentes).
