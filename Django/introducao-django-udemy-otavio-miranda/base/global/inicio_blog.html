Usando templates em nivel global

Estamos criando uma aplicação base para reutilização de codigo, é uma maneira de não ter que escrever as mesmas coisas varias vezes para cada pagina que precisar criar dentro do site

dentro da pasta base temos uma outra chamada global, onde guardamos os arquivos base do projeto.

Na pasta global, podemos adicionar cabeçalhos, menus e etc. dividida em partes, o que facilita e muito a organização geral da estrutura da pagina e de arquivos, além de que por exemplo, podemos usar um design para uma pagina inicial, uma pagina de blog ou qualquer outra coisa, e reciclar essas mesmas partes para criação de novas paginas


Exemplo do blog

Queremos que a pagina do blog, seja um pouco diferente da pagina inicial.

podemos criar um arquivo html para definir a estrutura base do html para o blog, e ao precisar extender as rotas dentro do blog, podemos usar esse mesmo arquivo incluindo ele diretamente no html da nova pagina


Observe:

nesse projeto criei dentro da pasta base/global um arquivo chamado inicio_blog.html, que é onde está o texto atual, vamos usar a mesma parte de cabeçalho do arquivo head ( como sabemos que se trata de pequenas partes, é bom criar uma pasta dentro de globals com o nome partial para melhor organização, e como escrito na propria documentação, isso é para nossa sanidade rsrsr.)

Feito isso podemos afirmar que o arquivo de cabeçalho esta no seguinte caminho base/global/partial/head.html

Agora podemos incluir esse mesmo cabeçalho para todas as paginas sem ter ficar escrevendo ele o tempo todo usando o metodo include.


Sintaxe

com o django html selecionado como linguagem basta apenas digitar include, e vai aparecer isso na linha.
{ % include "caminho/que/vai/ser/incluido" % }

Isso é muito parecido com a ideia de herança, onde a nova pagina vai carregar o conteudo que está sendo incluido na pagina
